# July 2026

## 2026-07-05 - PER-343 SD-card safety fixture

Objective: create a recoverable local test fixture before the app reads camera
media.

Method: detected `/Volumes/Untitled/DCIM`, then ran
`python3 scripts/copy_last_24h_media.py --source /Volumes/Untitled/DCIM --destination .local-test-data/sd-last-24h-20260705-1748 --hours 24`.
The helper copies into the local workspace, preserves relative `DCIM` paths, and
writes `reproduction.txt` plus `copy_manifest.json`.

Observations: initial copy included macOS AppleDouble `._*` files because they
share media extensions. The helper was corrected to skip names starting with
`._`, and the fixture manifest was regenerated.

Result: `.local-test-data/sd-last-24h-20260705-1748/` contains 477 real camera
files, 29,698,155,690 bytes: 216 JPG, 216 RAF, and 45 MOV. No `._*` files
remain.

Interpretation: the fixture is suitable for app testing while preserving the SD
card as the recoverable source. See
`decisions/agent/pending/PER-343-test-fixture-window-mtime.md` for the
mtime-window assumption.

## 2026-07-05 - PER-343 copied-fixture smoke test

Objective: validate the scanner and exporter against copied real camera media,
not synthetic files and not the live SD card.

Method: ran
`SDREVIEW_FIXTURE="/Users/vishaljain/Documents/Video editor/.local-test-data/sd-last-24h-20260705-1748" swift test --filter FixtureSmokeTests`.

Observations: the first run returned zero scanned items because the scanner used
`FileManager`'s hidden-file skipping option and the fixture lives under the
hidden `.local-test-data/` directory. The scanner now uses explicit media
extension filtering and skips only AppleDouble `._*` files.

Result: the smoke test passed. It scanned the copied fixture and exported one
real JPG plus one real MOV passthrough segment to
`.local-test-data/smoke-export-core/`; `manifest.json` had an empty failures
array.

Interpretation: the app core can read the copied camera data and write a
recoverable export without touching the source card.
